git和GitHub的使用（五）

GitHub使用练习

邀请某人成为团队成员

如果在另一个用户的本地仓库想要将克隆好的文件进行修改，并且推送到我的远程库中，需要我邀请他为团队成员才可以。

如果没有成为团队成员，进行推送的话将会失败。

如何邀请

在GitHub仓库中有一个setting设置

将邀请链接复制

将链接发送给你要邀请的人，对方访问邀请地址，接受即可。

远程库修改的拉取

另一成员对远程库中的内容进行修改后，我的本地库中并没有修改。

远程库

本地库

通过命令：git fetch origin master

可以将远程库抓取下来

尽管抓取下来了，但是并没有权力去改变本地库原来的文件，需要新命令去切换分支

命令：git checkout origin/master

现在就能发现改变了

现在的分支，查看分支命令：git branch -v

可以对分支进行合并，命令：git merge origin/master

协同开发时冲突的解决

要点

  • 如果不是基于GitHub远程库的最新版所做的修改，不能推送，必须先拉取。
  • 拉取下来后如果进入冲突状态，则按照“分支冲突解决”操作解决即可。

一起学习，一起进步 -.- ，如有错误，可以发评论
